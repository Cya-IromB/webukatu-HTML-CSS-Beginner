    thは見出しの定義、tdはデータ

metaタグ

  • 【meta charset="UTF-8"】
  • 【meta name="keywords" content=""】->カンマ区切りで検索キーワードを列挙する
  • 【meta name="description" content=""】->そのサイトの説明
  • 【meta name="robots" content=""】-> 検索ロボットにどの程度の許可を与えるか記述する。複数の命令を与える場合「,」で区切る。
    • 【noindex】 -> Googleなどの検索結果に表示して欲しくない時に使う。
    • 【nofollow】 -> robotにリンク先へ行くことを許可しない。
    • 【noarchive】-> キャッシュに保存してほしくない時に使う。使い所は常に最新の状態に在ることが望まれるページ。
  • 【meta name="author" content=""】-> その文章の作者を表す。ただの著名
  • 【meta name="generator" content=""】-> 何の文章自動生成ツールで作られた文章なのかを著名する。
  • 【meta http-equiv="refresh" content="n; url=xxxx.html"】-> n秒後、xxxx.htmlのページへ移行
  • 【meta http-equiv="refresh" content="n"】-> n秒後、再読み込み

    • w3.orgによれば、 meta要素によるrefreshを多用するウェブサイトは、検索エンジンのリストから削除されることがあるらしい。 それを避けたい場合には、代わりに301リダイレクトを使用する。

  • 【meta name="application-name" content=""】->そのページがウェブアプリケーションである場合、これを記述する。

疑似クラス

できること基本

  • 前後に何かくっつける
  • リストの装飾で特定行だけ指定できる
  • floatの解除

-疑似クラス一覧-

分類 クラス 説明
動的 :link 未訪問のリンク
:visited 訪問済みリンク
:active クリックされた
:hover マウスポインタが乗っている
:focus フォーカスされた
構造 :first-child 最初の要素(親要素の種類は無関係)
:last-child 最後の要素
:first-of-type 親要素と同一種類の要素のうち、最初の要素
:last-of-type 親要素と同一種類の要素のうち、最後の要素
:nth-child(an+b) 最初からan+b番目の要素。(odd)で奇数、(even)で偶数。
:nth-last-child(an+b) 最後からan+b番目の要素
:nth-of-type(num) 親要素と同一種類の要素のうち、最初からan+b番目の要素
:only-child ひとつだけの要素
:only-of-type 親要素と同一種類の要素のうち、ひとつだけの要素
:root html全体を指定する(要るのかこれ…)
:empty 空の要素(タグ間に何もない要素)を指定
:blank 空の要素(タグ間に何もない要素*空白は可)を指定
aタグと併用 :targer ＜a href="#x"＞から＜a id="x"＞へ移動した時、移動先の要素に作用する。
UIの状態 :enabled disabled属性が指定されていない
:disabled disabled属性が指定されている
:checked チェックされている(*フォームそのものには適用されないので、labelタグ等と併用する)
:indeterminate JavaScriptでindeterminate属性がtrueになっている
